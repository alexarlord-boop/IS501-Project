# Case Study: FreshBakes Baker
(in groups of 2-4 students)

FreshBakes Bakery is a family-owned shop in Nicosia, Cyprus, specializing in fresh pastries,
cakes, and custom orders. Currently, they handle walk-in sales, phone orders, and basic
delivery via manual records in notebooks and spreadsheets. With growing demand for
online orders (especially post-2025 tourism boom), the owner wants a web-based system
to manage customer orders, inventory, payments, and delivery scheduling. Key issues
include stockouts during peaks, order mix-ups, and no customer loyalty tracking. The
system must support 100+ daily orders, integrate simple payments (e.g., via Stripe), and
provide admin dashboards for bakers and delivery staff.

## Activity 1: Evaluate Methodologies
Discuss and critically evaluate the strengths and weaknesses of traditional (e.g., Waterfall)
and agile (e.g., Scrum) systems analysis methodologies. Compare them by referring to the
proposed web system for FreshBakes Bakery—e.g., how Waterfall suits fixed inventory
rules but Agile handles evolving custom cake designs better.

## Activity 2: Feasibility Study
Conduct a feasibility study for the web-based system for FreshBakes Bakery and produce a
feasibility report covering technical (e.g., cloud hosting viability), economic (e.g., €5,000
setup vs. €20,000 annual savings), operational (staff training), and schedule feasibility.
Evaluate the importance of these criteria for the bakery's context.

## Activity 3: Assess Methodologies
Review the user (e.g., easy order placement) and system requirements (e.g., real-time stock
updates) for the FreshBakes Bakery web system using a suitable methodology (justify Agile
for its iterations). Assess its effectiveness with examples, such as prototyping delivery
routes.

## Activity 4: System Design Specification
Design a system specification for the web-based FreshBakes Bakery system to meet user
and system requirements. Assess the design's effectiveness and methodology against
requirements.

## You are encouraged to include:
- Data Flow Diagrams (context, levels) showing order flow from customer to
baker/delivery.
- Use Case Diagram (actors: Customer, Baker, Admin, Delivery Staff; cases: Place
Order, Check Inventory, Update Stock).
- Entity Relationship Diagram (entities: Customer, Order, Product, Inventory,
Payment).
- Flowcharts (e.g., order fulfillment process).
- Prototypes (wireframes for order form, dashboard, mobile delivery view).
- Release plan (Phase 1: MVP ordering; Phase 2: Payments; tools: e.g. Figma for
prototypes, UML tools)
